# Сервер на Ubuntu: неочевидные возможности

## Кейсы энтузиастов

Многие администраторы используют Ubuntu Server далеко за пределами стандартных веб-серверов. Вот несколько интересных примеров.

| Проект | Оборудование | Что сделано | Технические детали |
|--------|--------------|-------------|---------------------|
| Домашняя облачная ферма | 3 старых ноутбука | Кластер `Kubernetes` для рендера видео | `k3s` + `longhorn`, потребление ~45 Вт |
| Метеостанция с ИИ | `Raspberry Pi 5` + датчики | Предсказание осадков по давлению | `Python`, `TensorFlow Lite`, `InfluxDB` |
| Pi-hole на стероидах | `Orange Pi Zero 2` | DNS + `unbound` + блокировка телеметрии | `docker-compose`, резервирование через `keepalived` |
| Сервер для умного дома | Intel NUC | `Home Assistant`, `Zigbee2MQTT`, `Node-RED` | Всё в LXC-контейнерах (`Proxmox`) |

## Зачем это вообще нужно?

> Один пользователь Reddit собрал кластер из 5 Raspberry Pi Zero 2 W, подключив их к одному блоку питания через `USB hub`. На этом кластере он запустил `distributed Nginx` + `GlusterFS`. Нагрузка — 2000 запросов/сек при энергопотреблении **менее 15 Вт**.

## Сравнение подходов к серверным задачам

| Задача | Ubuntu Server (x86) | Raspberry Pi (ARM) | Что интересного |
|--------|---------------------|--------------------|------------------|
| Файловое хранилище | `ZFS` + `Samba` | `NFS` + `USB 3.0` | На Pi можно добавить **RAID 0 через два SSD** |
| Медиасервер | `Jellyfin` + Intel QuickSync | `Jellyfin` + аппаратное декодирование | Pi 5 тянет **4K H.265** без проблем |
| Мониторинг | `Prometheus` + `Grafana` | `Prometheus` на голом `ARM` | Пишут кастомные экспортёры на `Go` |
| VPN-шлюз | `WireGuard` + `ufw` | `WireGuard` + `iptables` | Пропускает до **850 Мбит/с** на Pi 5 |

## Неочевидные проекты на Raspberry Pi с Ubuntu Server

*   **`rpi-clone`** — скрипт для горячего клонирования SD-карты на вторую карту. Позволяет сделать *горячую замену* при отказе.
*   **Автономный планировщик задач** — некоторые используют Pi как `cron`-хост для запуска `Ansible`-плейбуков в полях без интернета.
*   **`Pi-KVM`** — программный KVM-over-IP на базе Pi. Управление сервером через BIOS, даже если основная ОС не загружена.

## Выводы

| Что выбрать | Если нужно | Почему |
|-------------|------------|---------|
| **Ubuntu Server на x86** | Высокая производительность, `ZFS`, `KVM`, много дисков | Стабильность и зрелость репозиториев |
| **Ubuntu Server на Raspberry Pi** | Энергоэффективность, компактность, GPIO | `arm64` + официальная поддержка `Snap` и `Docker` |

> Некоторые ставят Ubuntu Server на Pi 4 и используют его как **роутер с AdGuard Home** и `SQM` для борьбы с буферизацией. Другие поднимают `GitLab Runner` на Pi для CI/CD в небольших проектах.
