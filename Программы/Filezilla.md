# FileZilla: FTP-клиент, который не пытается быть умным

## Что это такое

FileZilla — бесплатный кросс-платформенный FTP-клиент с открытым исходным кодом. Поддерживает FTP, FTPS (FTP с SSL/TLS) и SFTP (SSH File Transfer Protocol). Главная фишка — простота и скорость. Двухпанельный интерфейс, где слева ваш компьютер, справа сервер, и файлы летают туда-сюда перетаскиванием. Никаких анимаций, никаких помощников — просто работа.

## Поддерживаемые протоколы

| Протокол | Порт по умолчанию | Шифрование | Когда использовать |
|----------|-------------------|------------|--------------------|
| `FTP` | 21 | Нет (пароли в открытом виде) | Только в локальной сети |
| `FTPS (Implicit)` | 990 | TLS/SSL | Наследие, почти не используется |
| `FTPS (Explicit)` | 21 | TLS/SSL | Защищённый FTP, пароли шифруются |
| `SFTP` | 22 | SSH | Лучший выбор для современных серверов |
| `FTP over HTTP` | 80/443 | Зависит от HTTP | Экзотика, через браузер |

> **Важно**: обычный FTP передаёт логин и пароль в открытом виде. Используйте SFTP или хотя бы FTPS.

## Интересные сценарии использования

*   **Синхронизация с удалённым сервером как `rsync`** — в FileZilla есть функция `Синхронизированный просмотр` (View → Synchronized Browsing). При переходе в любую папку на локальной машине, FileZilla автоматически переходит в ту же папку на сервере. Удобно, если структура каталогов совпадает. Плюс включите `Сравнение директорий` (View → Directory Comparison) — и разноцветные файлы покажут, где расхождения.

*   **Автоматическая загрузка скриншотов** — на Windows можно настроить `AutoIt` скрипт, который при появлении нового файла в `%USERPROFILE%\Desktop\Screenshots` вызывает `filezilla.exe -c "0/Мой сайт"` и загружает его. В MacOS для этого есть `Hazel`.

*   **Прямое редактирование файлов на сервере** — правой кнопкой на файле → `Просмотреть/править`. FileZilla скачает файл во временную папку, откроет в программе по умолчанию (например, Notepad++). При сохранении спросит: загрузить обратно на сервер? Да — и файл обновлён без лишних движений.

*   **Обход блокировки FTP по порту 21** — на некоторых сетях (общежития, офисы) блокируют стандартные порты. В FileZilla можно указать нестандартный порт в строке подключения: `sftp://myserver.com:2222`. Если сервер настроен на порт `2222`, соединение пройдёт.

## Установка и первая настройка

```text
1. Скачать с официального сайта filezilla-project.org (избегайте зеркал с рекламой)
2. Установить. Внимание: в установщике предлагают дополнительное ПО (AVG, Opera). Снимите галочки.
3. Запустить. Откроется менеджер сайтов (File → Site Manager)
4. Нажать «New Site», ввести:
   - Protocol: SFTP (или FTPS)
   - Host: ip-адрес или домен
   - Port: 22 (для SFTP) или 21 (для FTP)
   - Logon Type: Normal или Ask for password
   - User: ваш логин
   - Password: сохранять или нет
5. Connect — и вы на сервере
```

### Настройки, которые стоит изменить сразу

| Меню | Параметр | Рекомендация |
|------|----------|--------------|
| Edit → Settings → Transfers | Maximum simultaneous transfers | `5` (не больше 10, чтобы не положить сервер) |
| Edit → Settings → Transfers | Limit download/upload speed | Поставьте `1000` KB/s, если канал общий |
| Edit → Settings → Interface | Show directory comparison tooltip | Включить |
| Edit → Settings → File editing | Default editor | Указать `Notepad++` или `VS Code` вместо встроенного |
| Edit → Settings → Updates | Check for updates | Оставить включённым (безопасность) |

## Проблемы и их решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| `ECONNREFUSED` | Сервер не слушает порт | Проверьте порт, включён ли FTP-сервер |
| `Connection timed out` | Файрволл или NAT | Откройте порт в брандмауэре, используйте пассивный режим |
| `500 Illegal PORT command` | Активный режим не работает | Настройки → FTP → Passive Mode (включить) |
| `Server sent passive reply with unroutable address` | Сервер отдаёт внутренний IP | Настройки → FTP → Passive Mode → Use the server's external IP address |
| `Failed to retrieve directory listing` | Проблема с листингом | Настройки → FTP → Transfer Mode → Binary (не ASCII) |
| Пароль не сохраняется | Мастер-пароль не установлен | Edit → Settings → Security → Set master password |

## Недостатки и ограничения

*   **Рекламное ПО в установщике** — официальный установщик предлагает дополнительное ПО, нужно снимать галочки вручную. Многие скачивают с зеркал и получают майнеры.
*   **Устаревший интерфейс** — выглядит как программа из 2005 года. Не адаптируется под тёмную тему Windows.
*   **Нет встроенного редактора** — только вызов внешнего. После сохранения файла нужно подтверждать загрузку.
*   **Нет синхронизации настроек между компьютерами** — только ручной экспорт/импорт `filezilla.xml`.
*   **SFTP не поддерживает `ssh-agent` напрямую** — нужно указывать путь к приватному ключу в настройках сайта.
*   **История паролей хранится в открытом виде** — без мастер-пароля любой, кто откроет компьютер, увидит пароли в `sitemanager.xml`.

## Альтернативы, если FileZilla не нравится

| Клиент | Почему лучше | Почему хуже |
|--------|--------------|--------------|
| WinSCP | Интеграция с PuTTY, `ssh-agent`, скрипты на PowerShell | Только Windows |
| Cyberduck | Красивый интерфейс, интеграция с облаками (S3, B2, GCS) | Нет двухпанельного режима |
| `lftp` | Мощнейший скриптинг, зеркалирование, параллельные сегменты | Консоль, порог входа |
| ForkLift | Двухпанельный файловый менеджер + FTP в одном флаконе | Только Mac, платный |
| Total Commander | FTP как плагин, всё остальное — файловый менеджер | Платный, сложный |

## Безопасность: что нужно знать

1. **Не храните пароли на общем компьютере** — FileZilla хранит их в `sitemanager.xml` без шифрования, если не установлен мастер-пароль.

2. **Обязательно включите мастер-пароль** — Edit → Settings → Security → Set master password. После этого все сохранённые пароли шифруются.

3. **Используйте SFTP вместо FTP** — даже если сервер поддерживает только FTP, попробуйте поднять `openssh-sftp-server`.

4. **Не подключайтесь к неизвестным FTP-серверам** — анонимные FTP могут быть ловушкой.

5. **FileZilla Server отдельно** — есть версия для Windows, но она не обновлялась годами. Лучше использовать `vsftpd` (Linux) или `OpenSSH` для SFTP.

## Заключение

FileZilla — это рабочий инструмент. Он некрасивый, неинтеллектуальный, иногда тупящий с кодировками (русские имена файлов превращаются в кракозябры, если сервер не в UTF-8). Но он просто делает свою работу: перекидывает файлы туда-сюда быстро и надёжно. Веб-разработчики на Windows используют его годами, потому что он предсказуем. Настройки один раз сохранил — и забыл. Если вам нужно залить сайт на хостинг, скачать пачку логов или синхронизировать папки с сервером, FileZilla справится без танцев с бубном. Если хочется красоты и облаков — смотрите в сторону Cyberduck. Если нужны скрипты и автоматизация — осваивайте `lftp`. Но для ручной работы с FTP/SFTP FileZilla остаётся золотым стандартом уже почти 20 лет.
