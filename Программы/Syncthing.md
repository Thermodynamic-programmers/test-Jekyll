# Syncthing

## Что это такое?

Syncthing — это программа для непрерывной синхронизации файлов между устройствами в локальной сети или через интернет. Все данные передаются напрямую, без промежуточных серверов. Шифрование включено по умолчанию, ключи генерируются на каждом устройстве.

## Основные возможности

| Функция | Описание | Пример использования |
|---------|----------|----------------------|
| `P2P` синхронизация | Прямое соединение между устройствами | Синхронизация фото между телефоном и ноутбуком без Google Drive |
| Версионирование файлов | Хранение старых версий | Восстановление `document.docx` после случайного удаления абзаца |
| Игнорирование файлов | Исключение по маске | `*.tmp`, `Thumbs.db`, `node_modules/` |
| Шифрование | TLS 1.3 на всех соединениях | Даже провайдер не видит имена файлов |
| LAN-обнаружение | Автоматическое нахождение соседей | Работает без `mdns` через broadcast |

## Кейсы реальных пользователей

| Пользователь | Устройства | Сценарий | Настройка |
|--------------|------------|----------|-----------|
| Фотограф | ПК + ноутбук + NAS | `RAW`-файлы с камеры на все машины | `staging` папка с версионированием `simple` |
| Разработчик | 3 ПК + VPS | Синхронизация `dotfiles` и `ssh` ключей | Игнорирование `*.log` и `.git/` |
| Семья | 5 телефонов + ПК | Общие фото детей | Только отправка на телефонах (режим `send only`) |
| Студент | Ноутбук + десктоп + планшет | Конспекты `Obsidian` | Версионирование `staggered` каждые 5 минут |

## Установка на Ubuntu Server

```bash
# Добавление официального репозитория
curl -s https://syncthing.net/release-key.txt | sudo apt-key add -
echo "deb https://apt.syncthing.net/ syncthing stable" | sudo tee /etc/apt/sources.list.d/syncthing.list
sudo apt update
sudo apt install syncthing

# Запуск как службы
sudo systemctl enable syncthing@$USER
sudo systemctl start syncthing@$USER
```

Интерфейс доступен по адресу `http://localhost:8384`

## Настройка через веб-интерфейс

| Шаг | Действие | Важно |
|-----|----------|-------|
| 1 | Нажать `Add Remote Device` | Ввести ID устройства (показывается сверху) |
| 2 | Выбрать папку для синхронизации | Можно указать несколько |
| 3 | Настроить `Folder Type` | `Send & Receive`, `Send Only` или `Receive Only` |
| 4 | Указать `File Versioning` | `Simple` (хранить 5 версий) или `Staggered` (умное удаление) |

## Интересные сценарии

*   **Резервное копирование с телефона на NAS** — на телефоне ставится `Syncthing-Fork`, папка `DCIM/Camera` синхронизируется в режиме `Send Only`. На NAS включается версионирование. Удалённые с телефона фото остаются в архиве.

*   **Синхронизация игровых сохранений** — папки `~/.minecraft/saves` и `~/.local/share/Steam/userdata` синхронизируются между ПК и ноутбуком. Игнорируются кэш и логи.

*   **Обмен файлами с малознакомыми людьми** — создаётся временная папка с паролем (`untrusted` устройство). Другой пользователь не видит остальные ваши данные.

*   **Журналирование с нескольких Raspberry Pi** — каждая малины пишет логи в свою папку, а центральный сервер собирает их в `Receive Only`. На сервере настроен `cron` с `grep` для поиска ошибок.

## Сравнение с аналогами

| Программа | Способ синхронизации | Шифрование | Версионирование | Бесплатно |
|-----------|----------------------|------------|-----------------|-----------|
| **Syncthing** | P2P | ✅ TLS | ✅ 5 режимов | ✅ |
| Resilio Sync | P2P | ❌ (опционально) | ✅ платно | ❌ |
| Nextcloud | Клиент-сервер | ✅ | ✅ | ✅ |
| Dropbox | Облако | ✅ | ✅ 30 дней | ❌ |
| `rsync` + `cron` | По расписанию | ❌ (только SSH) | ❌ (ручной) | ✅ |

## Ограничения и нюансы

> **На Android** нужно разрешить работу в фоне и отключить оптимизацию батареи для Syncthing. Иначе синхронизация будет с задержками.

> **Файлы, открытые в других программах**, могут не синхронизироваться. Например, база данных `SQLite` или файл блокировки `*.lock`.

> **При одновременном изменении файла на двух устройствах** Syncthing создаёт конфликтную копию с именем `file (device's sync conflict - 2026-04-04).ext`. Это не страшно, но требует ручного разрешения.

## Продвинутые настройки

*   **Ограничение скорости** — в интерфейсе: `Actions` → `Advanced` → `Rate Limit`. Полезно при синхронизации через мобильный интернет.

*   **Использование релейного сервера** — если прямое соединение невозможно (оба устройства за `CGNAT`), трафик идёт через публичный релей. Данные всё равно зашифрованы.

*   **Запуск только при подключении к определённой WiFi** — через `systemd` условие: `ConditionPathExists=/run/network/ssid-HomeNetwork`.

*   **Уведомления через `ntfy.sh`** — при завершении синхронизации можно выполнить скрипт:

```bash
curl -d "Синхронизация завершена" ntfy.sh/syncthing-my-topic
```

## Заключение

Syncthing не пытается быть красивым. У него нет официального клиента для iOS (есть сторонний `Möbius Sync`), нет встрощего редактора документов, нет истории изменений с красивыми временными шкалами. Но он работает там, где другие падают: при обрыве соединения, при смене IP, при наличии тысяч мелких файлов. Программа не требует регистрации, не знает ваших паролей и не видит содержимое. Для тех, кто готов потратить 15 минут на настройку, это лучший способ забыть о флешках и облачных лимитах.
